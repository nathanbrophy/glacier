## Public Summary

<!-- **Public.** One paragraph in end-user voice. The canonical description for the site and README. -->

`cli` is Glacier's gold-standard CLI builder. You write a plain Go struct with a `Run(ctx context.Context) error` method and annotate its fields with doc-comment markers. The accompanying `glaciergen` tool reads those markers, discovers the command tree, and emits a single `zz_generated_cli.go` file that wires everything together — flag parsing, env-var binding, help text, subcommand routing, signal handling, and the branded banner. No `init()` calls. No global registries buried in framework internals. No reflection at runtime for the happy path. You hand back a `Run` result; Glacier handles exit codes, error formatting, and the lifecycle the same way on every OS.

## Mental Model

<!-- **Public.** The conceptual frame a developer should hold while using this. Mermaid diagrams welcome. Source for the "Concepts" page on the site. -->

The developer's mental model is a two-step loop:

1. **Write a command struct.** Give it a `Run(ctx) error` method. Annotate fields with `+glacier:*` markers in doc comments. That is your entire surface — the rest is generated.
2. **Run `glaciergen`.** The codegen tool discovers every type in your module that satisfies `cli.Command`, builds the parent-child tree from the markers, and emits `zz_generated_cli.go`. That file calls `cli.Register` on each command in dependency order.

At runtime the framework is a thin dispatch layer over stdlib `flag`. It reads the generated registration, resolves the argv path, binds env vars, runs validation hooks, installs signal handling, and calls `Run`. Nothing in the hot path touches reflection.

    end

    S --> D
    M --> P
    D --> E
    P --> E
    E --> R
    A --> R
    R --> B
    B --> H
    H --> X

**Key invariants.**

- A type is a command if and only if it has the exact method `Run(ctx context.Context) error`. The interface is unexported on purpose; users never name it.
- Every field on the struct that has a `+glacier:` marker becomes a flag. Fields without markers are ignored by the codegen; they are yours to use as dependencies injected before `Run`.
- The generated file is a pure Go source file. It compiles with the host module. It never imports `cli/gen` at runtime. Removing the generator does not break a built binary.
- `cli.Default` is the package-level `*App` used by generated code. Programmatic use may construct a named `App` independently. Both modes are supported; they do not share state.

### Three-tier placement

`cli` is a Tier-2 leaf package. It may import:
- Tier-0 kernel: `option`, `errs`, `log`, `assert` (test-only), `term`
- `internal/sigh` (signal wiring)
- stdlib only

`cli` must not import any other leaf (`mock`, `httpmock`, `httpc`) or any mid-tier package. This is enforced by `internal/laytest/layering_test.go`.

`cli/gen` is build-time tooling. It imports `golang.org/x/tools/go/packages` (approved per §23.3) plus stdlib. It must not be imported by the runtime `cli` package. Test row 59 (`TestImportSurfaceStdlibOnly`) enforces this invariant.

### Codegen workflow

```
1. Developer annotates struct fields with +glacier:* markers.
2. Developer runs `go run github.com/nathanbrophy/glacier/cmd/cligen ./...`
   (or installs cligen and runs `glaciergen ./...`).
3. cli/gen.Generate(Options{Pattern: "./...", Check: false}) is called.
4. discover.go: go/packages loads the module with LoadFiles|LoadImports|LoadTypes.
   Every package in the same module is inspected. Third-party packages are excluded
   (module-prefix check enforced; test row 101).
5. For each package, parse.go walks doc comments. Lines matching "+glacier:" are
   extracted; the per-marker grammar regex is applied (§23.8 table).
6. emit.go builds the command tree, topologically sorts by parent-child relationship,
   and emits zz_generated_cli.go under the user's package directory. All string
   literals pass through strconv.Quote (§23.8 invariant).
7. If --check: the emitter writes to an in-memory buffer, diffs against the on-disk
   file, and exits non-zero with a human-readable diff if stale. CI gate codegen-drift
   runs this on every PR.
```

### Banner integration

`banner.go` uses `//go:embed ../../assets/logo/banner.txt` to embed the kaomoji + block-character banner at compile time. The embedded bytes are pre-rendered once at `init` time into two byte slices: one with 24-bit ANSI gradient escapes (TTY path) and one in plain text (non-TTY / `NO_COLOR` path). Banner output on an invocation with no subcommand argument writes the appropriate slice to the configured writer. The pre-computation means banner rendering on the hot path costs at most one allocation (the slice copy to the writer).

### Signal handling

`cli` delegates signal wiring to `internal/sigh`. When `App.Run` is called, it installs a `context.CancelFunc` tied to `os.Signal` notifications for `syscall.SIGINT` and `syscall.SIGTERM` (Unix) or `os.Interrupt` equivalent (Windows). The cancel is called exactly once on the first signal. The handler waits for `Run` to return before draining the signal channel. On Windows, `CTRL_BREAK_EVENT` is mapped to the cancel via a build-tagged file; the cross-platform test (row 30) asserts this.

### FlagSource integration with conf

`cli` implements the `conf.Source` interface as `FlagSource`. After flag parsing, `App.Lookup(name string) (string, bool)` returns the string-serialized value of any registered flag. `FlagSource.Lookup` delegates to `App.Lookup`. This allows conf's layered-config pipeline to treat parsed CLI flags as a named source layer, ordered above env vars and below explicit overrides.

### Lifecycle

```
New(opts...)          — construct App; does not start signal handler
Register(cmd, opts...) — add command to internal registry; thread-safe; returns error
App.Run(ctx, argv)    — resolve path, bind flags, start signal handler, call Run(ctx)
App.Main(argv)        — Run + os.Exit on error; calls Help on ErrUnknownCommand
App.Close()           — cancel signal handler goroutine; flush any buffered output;
                        close owned resources; idempotent; returns errs.Join of sub-errors
```

`App.Close` must be called when the `App` is no longer needed (deferred in `main`, or in tests via `t.Cleanup`). Calling `Close` multiple times is safe and returns `nil` on all subsequent calls. Close does not call `os.Exit`.

## Schema

<!-- **Internal.** Go types with invariants stated as `// invariant: ...` comments on each field. -->

```go
// Command is the interface every CLI command must satisfy.
// It is intentionally unexported — users never reference it by name.
// The method signature is the compile-time contract.
type command interface {
    Run(ctx context.Context) error
}

// Stub is a do-nothing implementation of the command interface.
// Useful for parent nodes that exist only to group children.
type Stub struct{}

func (Stub) Run(_ context.Context) error { return nil }

// regConfig holds the options accumulated for a single Register call.
// All fields are zero-value safe; absent options use defaults.
type regConfig struct {
    // name is the command name as it appears in argv.
    // invariant: matches ^[a-z][a-z0-9-]{0,31}$ (enforced by Register)
    name string

    // parent is the dot-separated path to the parent command (may be empty for root).
    // invariant: each segment satisfies the name regex; no trailing dot.
    parent string

    // aliases are additional names that route to this command.
    // invariant: each alias satisfies the name regex.
    aliases []string

    // short maps flag names to a single-byte ASCII letter.
    // invariant: each key is an existing flag name; each value satisfies [a-zA-Z].
    short map[string]rune

    // envVars maps flag names to environment variable names.
    // invariant: each env var name satisfies ^[A-Z][A-Z0-9_]*$.
    envVars map[string]string

    // help maps flag names to override help strings.
    // invariant: no ANSI escape sequences allowed in values (§23.11).
    help map[string]string

    // root marks this command as the root of the command tree.
    // invariant: at most one command per App may have root==true.
    root bool

    // deprecated marks flags as deprecated with an optional message.
    deprecated map[string]string

    // required marks flags that must be set (from any source).
    required map[string]struct{}

    // choices constrains string flags to an enumerated set.
    // invariant: each choice element satisfies ^[a-z0-9_-]+$; max 32 choices per flag.
    choices map[string][]string

    // validate is a per-flag validation function called after binding, before Run.
    // invariant: each value satisfies func(string) error.
    validate map[string]func(string) error
}

// appConfig holds the options accumulated for a New call.
type appConfig struct {
    // version is the human-readable version string.
    // When empty, ReadBuildInfo is consulted at first use.
    version string

    // stdout is where help, banner, and version output are written.
    // invariant: non-nil after New (defaults to os.Stdout).
    stdout io.Writer

    // stderr is where error messages are written.
    // invariant: non-nil after New (defaults to os.Stderr).
    stderr io.Writer

    // logger is the structured logger for lifecycle events.
    // invariant: non-nil after New (defaults to slog.Default()).
    logger *slog.Logger

    // showBanner controls banner display on bare invocation.
    // Default true (§23.15: WithoutBanner() is the opt-out).
    showBanner bool
}

// App is the central registry and dispatcher.
// invariant: App is goroutine-safe after construction; all exported methods are safe
//            to call concurrently except Close, which must be called at most once
//            concurrently (subsequent concurrent Close calls return nil safely).
// invariant: once Run or Main has been called once to completion, the same App
//            instance may be reused for subsequent invocations (flags are re-parsed
//            per invocation; state is not accumulated across Run calls).
type App struct {
    mu       sync.RWMutex
    cfg      appConfig
    commands map[string]*entry   // keyed by dot-separated path
    aliases  map[string]string   // alias → canonical path
    closeOnce sync.Once
    cancel   context.CancelFunc  // signal handler cancel; nil before first Run
    closed   chan struct{}
}

// FlagParseError is returned when argv contains a flag whose value cannot be
// coerced to the declared type (e.g. --port abc for an int flag).
// Replaces the former ParseError name (§23.15 rename).
// invariant: Name is the flag name without leading dashes; Err is the underlying
//            strconv or type-coercion error.
type FlagParseError struct {
    Name string
    Err  error
}

func (e *FlagParseError) Error() string { ... }
func (e *FlagParseError) Unwrap() error { return e.Err }

// ErrUnknownCommand is returned by Run/Main when argv names a command path that
// has no registration.
type ErrUnknownCommand struct{ Path string }

// ErrUnknownFlag is returned when argv contains an unrecognized --flag.
type ErrUnknownFlag struct{ Name string }

// ErrMultipleRoots is returned by Register when more than one command is marked root.
type ErrMultipleRoots struct{ First, Second string }

// ErrUnresolvedParent is returned by Register when the declared parent path does
// not correspond to any registered command.
type ErrUnresolvedParent struct{ Child, Parent string }

// ErrCancelled is returned by Run when the context was already cancelled before
// the handler was invoked.
var ErrCancelled = errors.New("glacier/cli: context cancelled before dispatch")

// ErrPanic wraps a value recovered from a handler panic.
// invariant: Value is the exact value passed to panic().
type ErrPanic struct {
    Value any
    Stack []byte // runtime.Stack output
}
```

### Registration

```go
// Register adds cmd to the App's command registry.
//
// cmd must have method Run(ctx context.Context) error; any other type causes a
// registration error with a descriptive message.
//
// opts configure the command's name, parent path, aliases, flag metadata, and
// root status. See the With* family below.
//
// Register is goroutine-safe; multiple goroutines may call Register concurrently
// during the construction phase (before the first Run call).
//
// Error contract:
//   - returns *ErrMultipleRoots if a second root is registered.
//   - returns *ErrUnresolvedParent if the declared parent path has no registration
//     at the time Register returns (parents must be registered before children).
//   - returns a descriptive non-nil error if cmd lacks Run(ctx) error.
//   - returns a descriptive non-nil error on duplicate name/path.
func (a *App) Register(cmd any, opts ...option.Option[regConfig]) error
```

### Dispatch

```go
// Run dispatches argv to the matching command and calls its Run(ctx) method.
//
// argv is the raw argument slice (typically os.Args[1:]).
// The first element that does not begin with "-" and is not a flag value is
// treated as the command name; subsequent elements are flags for that command.
// Subcommand depth is unlimited; each level consumes one argv segment.
//
// Automatic flags added to every command:
//   --help / -h   print usage and return nil
//   --version / -v  print version string and return nil (root command only)
//
// Signal handling: Run installs SIGINT/SIGTERM (Unix) or os.Interrupt (Windows)
// cancellation on ctx. The first signal cancels the derived context passed to the
// handler. Run waits for the handler to return.
//
// Error contract:
//   - returns *ErrUnknownCommand when argv names an unregistered path.
//   - returns *ErrUnknownFlag when argv names an unregistered flag.
//   - returns *FlagParseError when a flag value cannot be coerced.
//   - returns ErrCancelled if ctx is already done before dispatch.
//   - returns *ErrPanic if the handler panics (stack trace included).
//   - returns the handler's error value verbatim otherwise.
//
// Concurrency: goroutine-safe. Multiple goroutines may call Run concurrently on the
// same App; each invocation is independent. Signal handlers are per-invocation.
//
// Blocking: Run blocks until the handler returns or the signal-derived context is done.
func (a *App) Run(ctx context.Context, argv []string) error

// Main calls Run with os.Args[1:] and, on a non-nil error, formats the error
// to stderr and calls os.Exit(1). On success it calls os.Exit(0).
// On ErrUnknownCommand, Main also calls Help before exiting.
// Main never returns to the caller.
func (a *App) Main()

// Help writes the usage page for the command identified by path to the App's
// configured stdout writer. If path is empty, the root command's usage is written.
// Returns nil if the command exists, *ErrUnknownCommand otherwise.
func (a *App) Help(path string) error

// Version writes the version string (from WithVersion or ReadBuildInfo) to stdout.
func (a *App) Version()

// Banner writes the banner art to stdout. On a TTY the 24-bit ANSI gradient is used;
// on a non-TTY or when NO_COLOR / GLACIER_NO_COLOR is set, plain text is used.
// Banner is called automatically by Main on bare invocation (no subcommand).
// (§23.15: WithoutBanner() suppresses automatic banner; this method remains callable.)
func (a *App) Banner()

// Lookup returns the string-serialized value of the named flag as set during the
// most recent Run call. Returns ("", false) for unknown names.
// Lookup implements the conf.Source interface as FlagSource.
// Concurrency: goroutine-safe (reads under read-lock after Run returns).
func (a *App) Lookup(name string) (string, bool)

// Close shuts down the App: cancels the signal handler, flushes buffered output,
// and closes any owned resources. Close is idempotent — subsequent calls return nil.
// Errors from sub-close operations are merged via errs.Join. (§23.16)
// Concurrency: goroutine-safe.
func (a *App) Close() error
```

### Option constructors for Register (regConfig) — §21.9 + §23.6 additions

```go
// WithName sets the command name as it appears in argv.
// Precondition: name must match ^[a-z][a-z0-9-]{0,31}$.
// If not set, the type name is lowercased and used (with any trailing "Cmd" suffix removed).
func WithName(name string) option.Option[regConfig]

// WithParent sets the dot-separated path to the parent command.
// Example: WithParent("root.deploy") parents the command under the "deploy" subcommand
// of the root.
// Precondition: each segment of path must satisfy the name regex.
// Error: Register returns *ErrUnresolvedParent if path has no registration.
func WithParent(path string) option.Option[regConfig]

// WithAlias adds an additional name that routes to this command.
// Multiple WithAlias calls accumulate. Each alias must satisfy the name regex.
func WithAlias(alias string) option.Option[regConfig]

// WithRoot marks this command as the tree root.
// The root command receives --help and --version automatically.
// Only one command per App may be root; Register returns *ErrMultipleRoots on conflict.
// (§23.6 addition)
func WithRoot() option.Option[regConfig]

// WithFlagShort registers a single-character shorthand for the named flag.
// Precondition: name is a field name on the command struct (validated at Register time).
// Precondition: short satisfies [a-zA-Z] (single ASCII letter).
// Violation of the short constraint returns a registration error. (§23.6 addition)
func WithFlagShort(name string, short rune) option.Option[regConfig]

// WithFlagEnv binds the named flag to an environment variable.
// If the env var is set when Run is called, its value is used as the flag default.
// Argv still overrides env.
// Precondition: envVar satisfies ^[A-Z][A-Z0-9_]*$.
// Violation returns a registration error. (§23.6 addition)
func WithFlagEnv(name string, envVar string) option.Option[regConfig]

// WithFlagHelp overrides the godoc-extracted help string for the named flag.
// The help string must not contain ANSI escape sequences (§23.11); any such sequences
// are stripped silently before display.
// (§23.6 addition)
func WithFlagHelp(name string, help string) option.Option[regConfig]

// WithFlagRequired marks the named flag as required.
// Run returns a typed RequiredError if the flag is absent from all sources.
func WithFlagRequired(name string) option.Option[regConfig]

// WithFlagChoices constrains the named flag to an enumerated set of values.
// Precondition: each choice must satisfy ^[a-z0-9_-]+$; at most 32 choices.
// Run returns a typed ChoicesError if the provided value is not in the set.
func WithFlagChoices(name string, choices ...string) option.Option[regConfig]

// WithFlagDeprecated marks the named flag as deprecated.
// When a deprecated flag is used, a warning is logged to the App's logger before
// the handler is called. The flag still works. message may be empty.
func WithFlagDeprecated(name string, message string) option.Option[regConfig]

// WithFlagValidate registers a validation function for the named flag.
// The function is called after all flags are bound and env vars applied, before Run.
// A non-nil return aborts dispatch with the returned error.
func WithFlagValidate(name string, fn func(string) error) option.Option[regConfig]
```

### cli/gen package — Generate API

```go
// Options configures a Generate call.
type Options struct {
    // Pattern is the go/packages load pattern (e.g. "./...", "github.com/example/app/cmd/...").
    // Precondition: must be a valid go/packages pattern; non-empty.
    Pattern string

    // OutputName is the filename written under each discovered package directory.
    // Defaults to "zz_generated_cli.go".
    // Precondition: must be a simple filename (no path separators, no leading dots).
    OutputName string

    // AppName selects the named App to target in the generated code.
    // Defaults to "cli.Default". The +glacier:command app= marker overrides per-command.
    // Precondition: when non-empty, must be a valid Go identifier.
    AppName string

    // Check, when true, performs drift detection: Generate writes to an in-memory
    // buffer and diffs against the on-disk file. Returns a non-nil error with a
    // human-readable diff if stale. No files are written in check mode.
    Check bool

    // Lint, when true, upgrades unknown marker warnings to errors.
    Lint bool

    // Logger is used for warning/info messages during generation.
    // Defaults to slog.Default() when nil.
    Logger *slog.Logger
}

// Generate discovers cli.Command implementers matching opts.Pattern, parses
// their +glacier:* markers, and emits or checks the generated registration file.
//
// Generate is goroutine-safe for distinct opts.Pattern values. Concurrent calls
// with the same target package are serialized via a per-directory file lock to
// prevent torn writes.
//
// Error contract: returns a non-nil, descriptive error for:
//   - invalid or unresolvable Pattern
//   - marker grammar violations (per §23.8 table)
//   - multiple root commands
//   - parent-child cycle
//   - name collision (two commands at the same dotted path)
//   - output path safety violations (§23.8: .., absolute, UNC)
//   - drift detected when Check == true
//
// Idempotency: Generate(Generate(src)) produces byte-identical output.
// Determinism: output is stable across N invocations on the same source.
func Generate(opts Options) error
```

### §23.8 Codegen safety — marker character allowlist

The parser in `cli/gen/parse.go` applies a per-marker regex before any string is allowed into the code-generation pipeline. Strings that do not match the allowlist are rejected with a typed error; they are never passed to the emitter. This table is the authoritative grammar:

| Marker | Regex | Max length | Notes |
|---|---|---|---|
| `+glacier:command name=` | `^[a-z][a-z0-9-]{0,31}$` | 32 chars | Lowercase kebab; no leading digit; no trailing dash |
| `+glacier:command parent=` | `^([a-z][a-z0-9-]{0,31}\.)*[a-z][a-z0-9-]{0,31}$` | 128 chars total | Dot-separated segments each satisfying name regex; no `..`; no leading/trailing dot |
| `+glacier:command alias=` | `^[a-z][a-z0-9-]{0,31}$` | 32 chars | Same as name |
| `+glacier:command app=` | `^[A-Za-z_][A-Za-z0-9_]*$` | 64 chars | Valid Go identifier |
| `+glacier:default` | type-dependent: string payload via `strconv.Unquote`; numeric via `strconv.Parse*` | 256 chars raw | String payloads must be valid Go quoted strings; parsing rejects malformed input |
| `+glacier:short` | `^[a-zA-Z]$` | 1 char | Single ASCII letter only |
| `+glacier:env` | `^[A-Z][A-Z0-9_]*$` | 64 chars | Uppercase; no leading digit |
| `+glacier:required` | (no payload) | — | Presence marker; payload rejected |
| `+glacier:choices` | `^[a-z0-9_-]+(\|[a-z0-9_-]+){0,31}$` | 256 chars total | Pipe-separated; each element lowercase alphanum/dash/underscore; max 32 elements |
| `+glacier:deprecated` | `^[^\x00-\x1f"\\]{0,128}$` | 128 chars | Printable ASCII; no control chars, no quotes, no backslashes |
| `+glacier:validate` | `^[a-zA-Z_][a-zA-Z0-9_]*$` | 64 chars | Valid Go identifier; resolved via `go/types`; signature `func(string) error` enforced |
| `+glacier:root` | (no payload) | — | Presence marker; payload rejected |
| `+glacier:mock` | see spec 0012 | — | Governed by mock spec; cligen processes but grammar owned by 0012 |

### §23.8 strconv.Quote invariant

All string values flowing from marker payloads into generated Go source pass through `strconv.Quote` in `emit.go`. The emitter contains no `fmt.Sprintf("\"%s\"", ...)` or equivalent raw string-formatting patterns. Test rows 94 and 95 verify this two ways: (a) golden file byte-inspection; (b) static grep of `emit.go` source that fails the test if the forbidden pattern appears.

### §23.8 Identifier validation via go/types

Markers that reference Go identifiers (`+glacier:validate`) have their payload verified by loading the package with `go/types` and confirming the identifier resolves to a function with signature `func(string) error`. If the identifier does not resolve or has the wrong signature, codegen returns a typed error (test rows 87–89). This prevents generated code from referencing non-existent or mistyped functions.

### §23.8 Output path canonicalization

The emitter constructs the output file path through `internal/safefile`. All output paths are:
1. Resolved relative to the module root (absolute base from `go/packages`).
2. Canonicalized with `filepath.Clean`.
3. Checked for `..` components, leading absolute separators (Unix `/`), and UNC paths (`\\` on Windows).
4. Confirmed to sit under the module root.

Any violation returns a typed error before any file write. Test rows 96–99 verify the four cases.

### §23.9 Untrusted-input boundary — runtime cli

The runtime `cli` package accepts argv from the OS. The following limits are enforced before any processing:

- Individual argv elements longer than 64 KiB are rejected with a typed parse error (test row 62).
- Argv elements containing NUL bytes (`\x00`) are rejected with a typed parse error (test row 61).
- `FuzzArgvParse` (row 60) seeds the corpus with control-character-heavy and oversize inputs; any panic is a CI failure.

### §23.9 Untrusted-input boundary — codegen

The `go/packages` load mode is restricted to `LoadFiles | LoadImports | LoadTypes` — `LoadSyntax` is not included except where strictly required. This limits the AST surface exposed to the marker parser. Test row 129 asserts the LoadMode constant.

Cross-module discovery is blocked: `discover.go` enforces that every discovered package shares the host module path prefix. Test row 130 asserts this.

### §23.11 ANSI injection prevention in help text

`WithFlagHelp` strips ANSI escape sequences from the provided help string before storing it. The rendered help page never emits ANSI sequences injected via help text, regardless of source. This prevents a hostile `--help` invocation from emitting terminal control sequences via the help renderer. The emitter also escapes doc-comment text extracted for `WithFlagHelp` emissions in the generated file — any sequence matching `\x1b\[` is stripped before `strconv.Quote` is applied.

### Supply-chain posture

`golang.org/x/tools/go/packages` is imported only in `cli/gen`. The runtime `cli` package has zero external dependencies beyond the framework kernel. Test row 59 (`TestImportSurfaceStdlibOnly`) enforces this with a `go list -deps` assertion. The codegen binary (`cmd/cligen`) is a build-time tool; it never ships in the framework module's compiled output.

## FAQ

<!-- **Public.** Anticipated user questions with answers. Magpie extracts to the public docs FAQ. -->

**Q: Do I have to use codegen? Can I register commands programmatically?**

A: Codegen is optional. `cli.Register` accepts any value whose concrete type has `Run(ctx context.Context) error`. The `With*` option constructors give you full control over names, parents, aliases, flag metadata, and all marker equivalents. The generated file is a convenience, not a requirement. Many integration tests use programmatic registration directly.

**Q: How does the subcommand tree get depth beyond one level without code-gen?**

A: Call `Register` on the parent command first (it must be registered before its children), then call `Register` on the child with `WithParent("parentname")`. The parent path is a dot-separated sequence of command names from the root: `WithParent("root.deploy")` places the command under `root → deploy`.

**Q: What happens if my handler panics?**

A: `App.Run` recovers the panic and returns `*ErrPanic{Value: recovered, Stack: runtimeStack}`. `Main` formats the panic to stderr and exits non-zero. The stack trace is always captured. A build-tagged `glacier_debug` mode includes the stack in the formatted error display.

**Q: Why is the generated file named `zz_generated_cli.go` with the `zz_` prefix?**

A: The `zz_` prefix causes the file to sort last in directory listings and most IDEs, making it visually distinct from hand-written source. The `DO NOT EDIT` comment in the first line is machine-readable by `gofmt` and most editors. The naming convention is enforced by test row 118.

**Q: Does the banner render correctly in all terminals?**

A: The banner uses 24-bit ANSI color escapes on TTY output. When `NO_COLOR` or `GLACIER_NO_COLOR` is set to any non-empty value, or when the output is not a TTY, plain-text output is used (no escapes). `term.IsTTY(w)` (from spec 0016) is the detection mechanism. CI runs banner tests against a PTY emulator to verify ANSI presence.

**Q: Can I target a named App other than `cli.Default` with codegen?**

A: Yes. Add `+glacier:command app=MyApp` to the command struct (where `MyApp` must be a valid Go identifier in scope at the generated file's package). This causes the generated `Register` call to target `MyApp` instead of `cli.Default`. The `Options.AppName` field sets the module-wide default; the per-command `app=` modifier overrides per struct.

## Decisions & Rationale

<!-- **Internal.** Why-this-and-not-that for non-obvious choices. Folded-in ADR. -->

**D1 — Command interface is unexported.**
The `command` interface (the `Run(ctx) error` contract) is unexported. Users never reference it by name; they just implement the method. This eliminates the possibility of `interface{}` casts and keeps the public API surface minimal. The codegen tooling uses `go/types` to verify satisfaction without requiring the interface to be named.

**D2 — Separate runtime library and codegen library.**
`cli` (runtime) and `cli/gen` (codegen) are distinct packages. The runtime never imports `cli/gen`. This means a compiled binary does not carry the codegen dependency graph (`golang.org/x/tools/go/packages` and its transitives). Test row 59 locks this invariant. Users who don't use codegen pay no binary-size penalty.

**D3 — FlagParseError rename from ParseError (§23.15).**
The former `ParseError` name was ambiguous — it could mean command-name parse errors, marker parse errors, or flag value coercion errors. `FlagParseError` is unambiguous and scoped. The rename is a breaking change within proposed status (no implementations yet), so no migration path is required.

**D4 — WithoutBanner() as opt-out, banner on by default (§23.15).**
The original design had `WithBanner()` as an opt-in. §23.15 inverted this: banner is default-on because the brand experience is part of the CLI's first-run quality. Users who embed `cli` in a library context or who need quiet stdout suppress with `WithoutBanner()`. This avoids the situation where every new project forgets to call `WithBanner()`.

**D5 — golang.org/x/tools/go/packages approved for cli/gen (§23.3).**
The six Falcon questions are answered in the Dependency Justification table. The core argument: `go/packages` handles workspace mode, vendor, CGo, build constraints, and overlay filesystems. Building even a partial replacement in tree would be a multi-thousand-line undertaking with no quality ceiling. The Go team maintains it; it is not single-maintainer; it is BSD-3-Clause. The dep is isolated to the codegen package only.

**D6 — §23.6 additions: WithFlagShort, WithFlagEnv, WithFlagHelp, WithRoot.**
These four option constructors were missing from the §21.9 programmatic API. They were identified during the marker-to-option parity review in §23.6. Each marker (`+glacier:short`, `+glacier:env`) requires a corresponding programmatic option so that codegen-free users have full feature access. The `WithRoot()` constructor was the only way to mark a root programmatically without codegen.

**D7 — App.Close() and idempotency (§23.16).**
Signal handler goroutines and output buffers are owned resources. Without `Close`, tests that construct many `App` instances in parallel leak goroutines. `sync.Once` ensures idempotency. `errs.Join` ensures all sub-errors surface rather than the first one shadowing others. `Close` does not call `os.Exit` so that library consumers can use `App` without surrendering process control.

**D8 — Per-directory file lock for concurrent Generate calls.**
If two goroutines simultaneously call `Generate` targeting the same package directory, the naive implementation produces torn writes (partial files). The file lock (via `internal/safefile`) ensures only one writer at a time per directory. Callers targeting distinct directories run in full parallel (test row 123).

**D9 — Choices restricted to lowercase alphanum/dash/underscore.**
`+glacier:choices` is restricted to `^[a-z0-9_-]+$` per element. This avoids choices that look like flags (`--foo`), quote characters, or shell-significant tokens. The restriction is narrow enough that all realistic use cases (environment names, mode strings, log levels) fit within it.

**D10 — Marker payload NUL-and-newline rejection.**
A marker payload containing a NUL byte or a newline would be syntactically ambiguous in Go doc comments (newlines end the comment line; NUL is not valid Go source). The grammar rejects them before any processing. `FuzzMarkerParse` seeds specifically with NUL and newline inputs.

**D11 — +glacier:command app= modifier (§23.8 addition).**
Some modules declare multiple named `App` instances (e.g., one for the CLI and one for a daemon mode). Without the `app=` modifier, all codegen targets `cli.Default`. The modifier allows per-command targeting, enabling a single module to contain two independent command trees. The `Options.AppName` field provides the module-wide default.

**D12 — Generated file starts with `// Code generated by glaciergen; DO NOT EDIT.`**
This is the canonical Go generated-file header per `go generate` convention (documented in the Go spec and `go/generate` tooling). It signals to editors, linters, and code-review tools that the file is machine-produced and should not be manually edited. Test row 117 asserts the exact first line.

## Verification

<!-- **Internal.** Concrete steps to prove the change works end-to-end. Run when the spec moves to `verified`. -->

1. **Codegen drift gate.** CI job `codegen-drift` runs `go run ./cmd/cligen --check ./...` on every PR. Any stale `zz_generated_cli.go` causes a non-zero exit and blocks merge. This gate verifies §23.8 `--check` mode end-to-end. Test rows 111–113 cover the unit level; the CI gate covers the integration level.

2. **Marker fuzz.** CI job accumulates `FuzzMarkerParse` (row 125) for ≥ 1 hour on the PR-protect branch before every release tag. Any crash or panic is a release blocker. Corpus seeds include: NUL bytes, embedded newlines, shell-injection-shaped payloads, Unicode lookalikes for ASCII letters, oversize strings (> 256 bytes), and valid-looking payloads that differ by one character.

3. **Banner render.** Manual verification step: run `go run ./cmd/example-app` (shipped as `cli/gen/testdata/simple/`) in a PTY-capable terminal. Confirm the polar-bear kaomoji and GLACIER wordmark render with the correct gradient. Run the same binary with `NO_COLOR=1` and confirm plain text. Run in a pipe (`./example-app | cat`) and confirm no ANSI escapes. This covers §21.9 F7 / NF3 / F11 at the human-perceptible level beyond what the PTY unit test covers.

4. **`go test -race ./cli/... ./cli/gen/...`** passes on Linux, macOS, and Windows CI runners. Signal-handling tests (rows 28–30) honor build tags and assert only the OS-relevant assertions.

5. **Coverage thresholds.** CI runs `go test -coverprofile=cov.out ./cli/...` and asserts ≥ 90% line coverage. Separately for `./cli/gen/...` at ≥ 92%. 100% public-symbol coverage verified by the Lynx-owned public-API coverage audit script.

6. **Performance benchmarks.** `BenchmarkRunSimpleCommand` (row 63) must not exceed 5 µs/op on the CI baseline runner (per §23.13 recalibration). CI uses `benchstat` with N=5 runs and a 10% flake tolerance per §23.12. Regressions block merge.

7. **Import surface audit.** `TestImportSurfaceStdlibOnly` (row 59) passes: `go list -deps ./cli` contains no entry from `golang.org/x/tools/go/packages` or any other codegen dep. `cli/gen` is allowed its deps; `cli` is not.

8. **Cross-leaf Close audit.** Integration test `TestCloseAuditAcrossLeaves` (I11 in the integration matrix) constructs `App`, `mock.Mock[T]`, `httpmock.Transport`, and `httpc.Client`; closes each in arbitrary order; asserts all return nil on the second close. This verifies §23.16 idempotency across the full leaf tier.
